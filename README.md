## 3. Генерация базового TEI-документа

Перед запуском первого блока необходимо внести изменения в код:
Найдите строку с параметром `--doc-index` и замените значение `default` на номер вашего документа (номер можно посмотреть в манифесте файлов - это скорее индекс(начиная с 0)):

```python
parser.add_argument("--doc-index", type=int, default=5, help="Process only one specific document index (0-based).")
```

После этого запустите **Первый блок**. На выходе в папке `tei_reports` появится документ в формате `Tei(xml)`.

## 4. Обработка таблиц

Это наиболее трудоемкая часть процесса, требующая точности.

![Пример таблиц в PDF](img/table_example.png)

1. Найдите все таблицы в исходном PDF-документе.
2. **Сделайте скриншоты каждой таблицы и сохраните их отдельно - очень важный пункт!!!** 
3. Перейдите в [Google AI Studio](https://aistudio.google.com/).
4. Загрузите скриншоты таблиц и попросите нейросеть преобразовать их в формат **CSV**.
5. Создайте в Colab директорию `tables/` и загрузите туда полученные CSV-файлы.
6. В начале вашего TEI(xml) файла добавьте ссылки на таблицы в секции `<listBibl>`:

```xml
<listBibl type="csv-tables">
    <bibl xml:id="table_csv_table1">
        <ref target="tables/table1.csv" type="csv">table1.csv</ref>
    </bibl>
</listBibl>
```

7. Пройдите по всему тексту XML-файла и в местах, где должны быть таблицы, добавьте ссылки на них:

![Пример вставки ссылок на таблицы в начале](img/xml_table_links.png)

![Пример вставки ссылок на таблицы в самом тексте](img/xml_table_upon.png)

8. Проверьте выделенные сущности вручную.

## 5. Классификация сущностей (Type и Subtype)

1. Запустите **второй блок** кода, предварительно настроив пути. На выходе вы получите CSV-таблицу.
2. Загрузите в нейросеть текст самого файла tei(xml), выходную таблиу CSV и схему приложенную ниже и попросите нейросеть заполнить колонки `Type` и `Subtype`, опираясь на следующую схему:
    Люди (типы):
        Власть (Администрация / Начальство)
        Медицина
        Служащие (Чиновничество / Канцелярские служители)
        Сословия (Купцы и прочее)

    Организации (типы):
        Тюрьмы
        Приски
        Медицина
        Образование
        Гос. Органы
        Пром-ть
        Церковь
        Общ. орг-ии
        Войска

    Места (типы):
        Адм. единицы
        Насел. пункты
        Географические точки

![Схема классификации сущностей](img/schema.png)

Примерный промпт будет звучать так:
> "ознакомься с отчетом xml чтобы понимать контекст, также ознакомься с схемой типов(она в тексте). я тебе прикрепил csv файл, твоя задача его заполнить по шаблону "Тег;Текст сущности;REF (ID);Частота;Type (заполнить);Subtype (заполнить)", заполнять только поля type и subtype, твоя задача их размечать строго по тому как на картинке(гос организации и остальное это все разные типы), на выходе дай мне обратно полностью csv файл со всеми заполнеными полями".

Либо можно сделать по другому, для больше точности:
> "ознакомься с отчетом xml чтобы понимать контекст, также ознакомься с последняя.png(это как размечать типы по сущностям). я щас тебе буду скидывать разные сущности, твоя задача их размечать строго по тому как на картинке(гос организации и остальное это все разные типы)" - в данном сценарии вы не прикрепляете ему CSV-таблицу, а по 3-4 строки скидываете ему из вашего CSV файла, далее просто эти же строчки редактируете их у себя в файле. 

3. Заполненный файл загрузите обратно в Colab.

(промпт по типу: "я тебе скидываю дату, и ты из контекста придумываешь event-ref для этого и как это вставить в список StandOff. просто переписывай строчку уже с вписанным event-ref" здесь так же два сценария, либо сразу весь CSV файл(для этого требуется поменять промпт), либо построчно(что лучше)).
